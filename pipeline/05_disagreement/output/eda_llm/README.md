# EDA Results — LLM-Extracted Features cho dự đoán giá cà phê

> Generated by `pipeline/05_disagreement/03_eda_llm_features.py`
>
> Data: 1,097 ngày features (LLM extraction bằng DeepSeek V4 Flash), từ 2023-03-11 đến 2026-03-11

---

## Tổng quan pipeline

```
7,048 bài báo (crawled HTML)
   → LLM extraction (DeepSeek V4 Flash): is_coffee_price, direction, price_vnd, certainty
   → Join với DiscoverLinks metadata (TARGET, PUBLISHED_DATE, DOMAIN)
   → Aggregate theo ngày  →  1,097 ngày features × 60 cột
   → Join với giá thực (Investing.com)  →  ~755 ngày giao dịch
```

**So sánh với pipeline thống kê truyền thống:**

| Khía cạnh | Thống kê (regex) | LLM (DeepSeek) |
|-----------|-------------------|-----------------|
| Nguồn tín hiệu | TITLE+SNIPPET (Google) + CONTENT (HTML) | CONTENT (HTML) qua LLM |
| Direction | Đếm keyword tăng/giảm/ổn định | Classification UP/DOWN/STABLE/MIXED |
| Giá | Regex pattern VND | LLM extraction + validation |
| Tin cậy | Không có | `certainty` 1–5 |
| Cross-check | dir_score_ts vs dir_score_body | certainty + consistency check |

## Các kết quả chính

### 1. Coverage — Dữ liệu đủ dùng không?

**Figure:** `01_coverage_timeline.png`

LLM coverage thấp hơn regex (do chỉ xử lý bài có CONTENT, không dùng SNIPPET), nhưng vẫn đủ dùng. LLM thêm một tầng lọc: `is_coffee_price` xác định bài thực sự về giá cà phê.

| Chỉ số | Robusta | Arabica |
|--------|---------|---------|
| Bài/ngày (trung bình) | 3.3 | 3.6 |
| Bài coffee/ngày | 2.5 | 2.2 |
| Coffee detection rate | 75.4% | 61.6% |
| Nguồn/ngày | 3.5 | 3.8 |
| Ngày join được với giá | 757/766 (98.8%) | 752/761 (98.8%) |

**Ý nghĩa:** ~25–38% bài bị LLM loại vì không thực sự về giá cà phê — đây là tầng lọc mà regex không có, giúp giảm nhiễu.

### 2. Sentiment LLM vs giá thực — LLM có phản ánh đúng xu hướng?

**Figure:** `02_sentiment_vs_price.png`

Biểu đồ overlay dir_score (LLM) lên đường giá thực. Tương tự bên thống kê, sentiment LLM phản ánh xu hướng giá nhưng chủ yếu concurrent/lagging. Tuy nhiên, tín hiệu LLM sạch hơn do phân loại UP/DOWN/STABLE/MIXED thay vì chỉ đếm keyword.

**Ý nghĩa cho model:** dir_score từ LLM cho correlation mạnh hơn hẳn so với regex (xem Section 5).

### 3. Certainty Analysis — LLM tự tin đến đâu?

**Figure:** `03_certainty_analysis.png`

Thay thế Section 3 của bên thống kê (cross-check ts vs body). Đây là thế mạnh riêng của LLM — mỗi bài có điểm certainty (1=giá thực tế, 5=không có giá).

| Target | certainty_mean | pct_high_cert (certainty ≤ 2) |
|--------|---------------|-------------------------------|
| Robusta | 1.53 | 98.9% |
| Arabica | 1.52 | 99.2% |

Certainty rất cao (~99% bài có certainty ≤ 2), cho thấy LLM tự tin với phần lớn extractions. Biểu đồ 3b cho thấy mối liên hệ giữa pct_high_cert và price_cv (consensus giá).

**Ý nghĩa:** Có thể tin tưởng kết quả extraction của LLM. Trường certainty cho phép lọc thêm khi cần (ví dụ: chỉ dùng bài certainty ≤ 2 cho model).

### 5. Feature nào correlate mạnh nhất với return ngày sau?

**Figure:** `05_correlation_vs_return.png`

**Đây là kết quả quan trọng nhất — LLM features mạnh hơn hẳn regex.**

| Rank | Feature | Robusta r | Arabica r | So sánh với regex |
|------|---------|-----------|-----------|-------------------|
| 1 | `dir_score_lag1` | **+0.146** | +0.043 | Regex: +0.049 → LLM gấp 3x (Robusta) |
| 2 | `pct_up_lag1` | **+0.130** | +0.043 | Regex `pct_tang_d`: — |
| 3 | `pct_down_lag1` | **−0.114** | −0.062 | Regex `pct_giam_d_lag1`: −0.078 → LLM 1.5x |
| 4 | `dir_entropy` | −0.092 | +0.042 | Regex: +0.064 (Arabica) |
| 5 | `dir_score_roll3` | +0.090 | +0.051 | — |

**Highlight:** `dir_score_lag1` của LLM đạt r=+0.146 cho Robusta — correlation rất mạnh cho dự đoán tài chính hàng ngày. Bên regex chỉ đạt ~0.05–0.08 cho features tốt nhất.

**Ý nghĩa:** LLM extraction cho tín hiệu chất lượng cao hơn đáng kể so với regex, đặc biệt cho Robusta. Điều này xác nhận giá trị của việc dùng LLM thay vì đếm keyword.

### 6. Conditional returns — Signal nào thực sự tạo khác biệt?

**Figure:** `06_conditional_returns.png`

So sánh return trung bình ngày sau khi signal = 1 vs signal = 0.

**Robusta — Kết quả đáng chú ý:**

| Signal | Δ Return | Nhận xét |
|--------|----------|----------|
| SIG_BULLISH | Dương | LLM bullish → return dương |
| SIG_BEARISH | Dương (bất ngờ) | Cần kiểm tra thêm |
| SIG_HIGH_CONFIDENCE | — | Đa số bài đều high confidence |

**Arabica — Kết quả đáng chú ý:**

| Signal | Δ Return | Nhận xét |

| SIG_BULLISH | Dương | Đúng chiều |
| SIG_BEARISH | Âm | Đúng chiều, mạnh hơn Robusta |

**Ý nghĩa:** Signals LLM hoạt động đúng chiều cho Arabica. Robusta cần phân tích thêm do SIG_BEARISH cho kết quả bất ngờ.

## Features khuyến nghị đưa vào model (LLM)

Dựa trên EDA, danh sách features ưu tiên:

1. `dir_score_lag1` — sentiment tổng hợp hôm qua (**feature mạnh nhất**)
2. `pct_up_lag1` — tỉ lệ bài UP hôm qua
3. `pct_down_lag1` — tỉ lệ bài DOWN hôm qua
4. `dir_entropy` — mức bất đồng ý kiến
5. `dir_score_roll3` — sentiment trung bình 3 ngày
6. `price_cv` — disagreement giá
7. `price_change_median_lag1` — thay đổi giá median hôm qua
8. `certainty_mean` — LLM confidence (unique)

Các SIG_ signals hữu ích:

- `SIG_BULLISH`, `SIG_BEARISH` — direction chính
- `SIG_HIGH_ENTROPY` — bất đồng cao
- `SIG_HIGH_CONFIDENCE` — LLM tin cậy cao (unique)
